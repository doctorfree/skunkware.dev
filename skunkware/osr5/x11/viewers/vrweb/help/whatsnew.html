VRweb 1.5 Release Notes

New in VRweb 1.5

  • Editing functionality

  • Rendering transparent polygons

  • View pulldown menu with viewing (new: view selection, world) and selection functions

  • select previous/next viewpoint with PgUp/PgDn

  • Mesa 2.1: faster flat shading on 8bit displays (dithered)

  • Polygon anti-aliasing (OpenGL only)

  • Passing VRML 2.0 files to a helper application

  • Bugfixes (nested WWWInline nodes), destination URLs (#viewpoint) etc.

New in VRweb 1.3.2

  • Stereo rendering in red/green mode

  • flip object: rotation about arbitrary point or (1.5) selected object

  • Documentation of Mouse and Key bindings (added Nov 28).

What's new in VRweb 1.3.1

  • Incorporation of Mesa 2.0

    • new, triangle-based renderer fixes bugs at polygon boundaries
    • faster texturing and smooth shading
    • support for texture antialising: see display options.
    • known bug: flat shading on 8bit visuals became slower
      (located and fixed in Mesa 2.1)

  • After korean now also japanese user interface

  • Bugfixes (BSP now also works on Linux/ELF and FreeBSD; again working unter X11R4)

What's new in VRweb 1.3

  • AsciiText/FontStyle

    VRweb is now able to render Text. First you have to install the font files as described in the installation guide.

  • Proxy Support

    WWWinline/Texture requests can be made via a proxy. See anchor setup.

  • Better VRML 1.0 conformance

    Non-convex faces are triangulated, vertex normal vectors are automatically generated (autosmoothing), face normal calculation is more robust. Many scenes will look much nicer now.

  • New rendering features

    Optionally, you can use a BSP renderer, which is faster (especially with shadow volumes), but consumes more memory and still has some bugs.

    Quadrics complexity (i.e. the no. of slices spheres, cones, and cylinders are split into) can be set interactively or via Xattribute.

    Some rendering issues concerning material settings have been improved. Scenes with frequent material changes will render faster (especially in the OpenGL version).
