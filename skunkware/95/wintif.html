Switching between Motif and Windows Look-and-feel

Motif applications can be recompiled and relinked with the header files and library provided on Skunkware. Assuming the Skunkware CD-ROM is mounted on /usr/skunk, recompile with the -I/usr/skunk/include compiler option and relink with the Wintif library /usr/skunk/lib/libWXm.a.

Applications relinked in this manner will only execute on systems with a valid Motif license.

By default, IXI Wintif 1.0 displays using the Microsoft windows 3.1 look-and-feel. However, if you wish to do so, you can switch to the Motif look. This can be done in two ways:

  • By setting the resource "lookAndFeel" to "Motif".
    For example:
    • When you start an application, add the following command-line argument:
      -xrm "*lookAndFeel: Motif"
    • In a resource file, add the line:
      *lookAndFeel: Motif
  • By setting the environment variable "WINTIF_LOOKANDFEEL" to "Motif".

If both the environment variable and the resource are set, the environment variable will take precedence.
