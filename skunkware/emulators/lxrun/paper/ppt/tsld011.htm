X11 DISPLAY Mapping

  • For local displays, SCO OpenServer uses “:0.0” while UnixWare uses “unix:0.0”

  • Former causes Linux X11 binaries to use shared memory, which won’t work with the native X server

  • Latter causes a lookup for machine “unix”

  • Lxrun remaps the DISPLAY appropriately
